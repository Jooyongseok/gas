## 1. 프로젝트 개요

### 무엇을 만들었나

전자코(E-nose)와 적외선 열화상 카메라 두 가지 센서를 동시에 사용해서 **가스 누출을 탐지하고 종류를 분류**하는 딥러닝 시스템이다.

### 왜 멀티모달인가

| 단일 센서 방식 | 멀티모달 방식 (본 프로젝트) |
|--------------|--------------------------|
| 하나의 센서만 사용 → 오탐 가능성 높음 | 두 종류의 센서 데이터를 동시에 학습 |
| 가스 농도나 온도 변화에 취약 | 서로 다른 물리적 특성을 상호 보완 |
| 낮은 정확도 | 최대 99.25% 정확도 달성 |

### 분류 대상 (4종)

| 클래스 | 레이블 | 설명 |
|--------|--------|------|
| Mixture | 0 | 혼합 가스 |
| NoGas | 1 | 가스 없음 (정상) |
| Perfume | 2 | 향수 (가스형 물질) |
| Smoke | 3 | 연기 |

### 주요 성과

- **Intermediate Fusion 방식**: 정확도 98.47%
- **Multitask Fusion 방식**: 정확도 99.25%
- 한국정보과학회 학부생/주니어 논문 경진대회 **31위** 수상

## 3. 파일별 상세 분석

---

### main.py

```python
from test import test

if __name__ == '__main__':
    test()
```

- 가장 단순한 진입점. `test.py`의 `test()` 함수를 호출하는 것이 전부다.
- 학습이 아닌 **추론/평가** 실행용 엔트리포인트.
- 학습을 하려면 `train.py`를 직접 실행해야 한다.

---

### train.py

학습 전체 흐름을 담당하는 핵심 스크립트.

**동작 순서:**
1. `GasDataSet`으로 train/test 데이터셋 로드 (CSV 파일 기반)
2. `DataLoader` 생성 (train: shuffle=True, test: shuffle=False)
3. `MultitaskFusionModel` 생성 및 GPU/CPU 배치
4. `CrossEntropyLoss` + `Adam(lr=1e-4)` 설정
5. 에폭마다 forward → loss → backward → gradient clipping → step
6. 에폭 끝마다 train loss / train accuracy 출력
7. 학습 완료 후 test 정확도 출력
8. 모델 가중치 저장: `Multitask_fusion_model.pt`

**주요 설정:**
| 항목 | 값 |
|------|-----|
| Batch Size | 8 |
| Epochs | 3 |
| Optimizer | Adam |
| Learning Rate | 1e-4 |
| Loss Function | CrossEntropyLoss |
| Gradient Clipping | max_norm=1.0 |

**참고:** 코드 상단에 `GasDataset`(폴더 기반) 방식이 주석 처리되어 있고, 현재는 `GasDataSet`(CSV 기반)을 사용한다.

---

### test.py

저장된 모델을 불러와서 테스트셋 정확도를 측정하는 스크립트.

**동작 순서:**
1. `GasDataSet`(TEST_CSV_PATH)으로 테스트 데이터 로드
2. `Multitask_fusion_model.pt` 가중치 로드 (`weights_only=True` — 보안 옵션)
3. `model.eval()` + `torch.no_grad()` 로 추론 모드 진입
4. `tqdm`으로 배치별 진행 상황 표시
5. 전체 정확도 `[Test Accuracy] XX.XX%` 출력

**버그 발견:** `print(start - time.time())`는 음수가 출력된다. `time.time() - start`로 작성해야 경과 시간이 올바르게 나온다.

```python
    c, _ = pywt.dwt(feature_data, 'haar')

    # 출력 차원 = 입력 차원 // 2
```

- **목적**: CNN 출력 특징에서 저주파(핵심 패턴) / 고주파(노이즈) 분리
- **Haar 웨이블릿**: 가장 단순한 웨이블릿, 평균(저주파) + 차이(고주파) 계산
- **출력**: 저주파 계수만 유지 → 차원이 절반으로 줄어듦
  - ResNet50: 2048 → 1024
  - Inception: 768 → 384
  - MobileNet: 1280 → 640
- **주의**: GPU 텐서를 CPU로 이동(`detach().cpu()`)해서 numpy 처리 후 다시 GPU로 돌려보내는 비효율이 있음

#### apply_dct (Discrete Cosine Transform)

```python
def apply_dct(x):
    result = dct(x_np, norm='ortho', axis=1)
    # axis=1: 특징 차원 방향으로 DCT 적용
```

- **목적**: 융합된 전체 특징 벡터를 주파수 도메인으로 변환
- **norm='ortho'**: 에너지 보존 정규화
- **효과**: 중요한 에너지가 저주파 성분에 집중 → BiLSTM 학습 효율 향상

### src/models/bi_lstm.py — 양방향 LSTM 분류기

```python
class BiLSTMClassifier(nn.Module):
    def __init__(self, input_dim, hidden_dim=256):
        self.lstm = nn.LSTM(input_dim, hidden_dim,
                            batch_first=True, bidirectional=True)
        self.dropout = nn.Dropout(0.3)
        self.classifier = nn.Linear(hidden_dim*2, NUM_CLASSES)
```

**forward 동작:**
```
입력 x (N, input_dim)
    → unsqueeze(1)          ← 시퀀스 길이 1로 만들기 (N, 1, input_dim)
    → BiLSTM                ← 순방향 + 역방향 LSTM
    → 마지막 타임스텝 출력    ← (N, hidden_dim*2) = (N, 512)
    → Dropout(0.3)
    → Linear(512 → 4)
    → 최종 로짓 (N, 4)
```

**왜 BiLSTM인가:** 원래 시계열 처리용이지만, 여기서는 융합된 특징 벡터(시퀀스 길이 1)를 처리하는 비선형 분류기 역할을 한다. 단순 MLP 대비 게이팅 메커니즘으로 중요한 특징을 선택적으로 활용한다.

### src/models/multitask_fusion_model.py — 핵심 통합 모델

전체 파이프라인을 하나로 묶는 메인 모델 클래스.

```python
class MultitaskFusionModel(nn.Module):
    def __init__(self):
        # 열화상용 CNN 3개
        self.cnn_thermal = nn.ModuleList([
            CNNFeatureExtractor('resnet50'),   # out: 2048
            CNNFeatureExtractor('inception'),  # out: 768
            CNNFeatureExtractor('mobilenet'),  # out: 1280
        ])
        # 센서용 CNN 3개 (동일 구조)
        self.cnn_sensor = nn.ModuleList([...])

        # DWT 후 차원: (2048//2 + 768//2 + 1280//2) × 2
        #            = (1024 + 384 + 640) × 2 = 4096
        total_dim = (2048//2 + 768//2 + 1280//2) * 2
        self.bilstm = BiLSTMClassifier(input_dim=total_dim)  # input=4096
```

**forward 단계별 차원 변화:**

```
[열화상 이미지] (N, 3, 224, 224)
    ├─ ResNet50   → (N, 2048) → DWT → (N, 1024)
    ├─ Inception  → (N,  768) → DWT → (N,  384)
    └─ MobileNet  → (N, 1280) → DWT → (N,  640)

[센서 히트맵] (N, 3, 224, 224)
    ├─ ResNet50   → (N, 2048) → DWT → (N, 1024)
    ├─ Inception  → (N,  768) → DWT → (N,  384)
    └─ MobileNet  → (N, 1280) → DWT → (N,  640)

→ torch.cat (6개 벡터 연결) → (N, 4096)
→ apply_dct                 → (N, 4096)  [주파수 변환]
→ BiLSTM                    → (N, 512)
→ Dropout(0.3)
→ Linear(512 → 4)
→ 최종 출력 (N, 4)           [Mixture / NoGas / Perfume / Smoke]
```

- 실행 불가능한 상태이며, 현재 사용되지 않는다.

## 8. 발전 방향 및 추후 기획

---

### 단기 개선 (1~2개월)

#### ① 버그 수정 및 코드 정리

- `test.py` 경과 시간 계산 수정
- `heat_transform.py`에 CSV 로딩 코드 추가

- `data/data_g.py` 함수명 추가 또는 파일 삭제

#### ② FastAPI 추론 서버 구축

현재는 스크립트 직접 실행만 가능하다. REST API로 래핑하면 실제 서비스 연동이 가능해진다.

```
POST /predict
  Body: { thermal_image: base64, sensor_values: [7개 수치] }
  Response: { class: "Smoke", confidence: 0.9872 }
```

#### ③ DWT GPU 연산 최적화

현재 CPU로 왕복하는 DWT를 PyTorch 텐서 연산으로 대체하면 학습 속도가 크게 개선된다.

---

### 중기 개선 (2~6개월)

#### ④ 모델 고도화 실험

현재 `extractors2.py`에 구현만 해두고 사용하지 않는 EfficientNet-B0, DenseNet-121, ConvNeXt-Tiny를 실제 모델에 적용해서 성능 비교 실험을 진행할 수 있다.

| 실험 | 예상 효과 |
|------|----------|
| extractors v2로 교체 | 더 가벼운 모델로 동급 성능 가능성 |
| v1 + v2 앙상블 (CNN 6개) | 정확도 추가 향상 가능 |
| Vision Transformer(ViT) 도입 | 전역 특징 추출 강화 |

#### ⑤ 실시간 IoT 파이프라인 구축

```
[MOS 가스 센서 7개]  →  [MCU/Raspberry Pi]  →  [Edge 추론]
[열화상 카메라]       →         ↓                     ↓
                       [데이터 수집 서버]      [경보 시스템]
                                                  ↓
                                          [모니터링 대시보드]
```

#### ⑥ 설명 가능 AI (XAI) 추가

- **Grad-CAM**: 열화상 이미지에서 모델이 어느 영역을 보고 판단했는지 시각화
- **SHAP**: 7개 센서 중 어떤 센서가 분류에 가장 기여했는지 수치화

---

현재 4종(NoGas, Smoke, Perfume, Mixture) → CO, NH3, H2S 등 산업용 위험 가스 클래스 추가
